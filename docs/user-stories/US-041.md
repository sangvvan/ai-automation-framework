# US-041 — Token budget guardrail

## Story
As a **framework operator**
I want to cap AI token usage per run
So that a misconfigured provider cannot rack up large bills.

## Acceptance criteria

### AC-1: Budget config
- **Given** `ai.tokenBudgetPerRun: 200000`
- **When** explore mode runs
- **Then** total tokens (input + output) across all AI calls cannot
  exceed the budget; the call that would breach raises
  `BudgetExceededError`.

### AC-2: CLI override
- **Given** `--budget 50000`
- **When** the run starts
- **Then** the value in config is overridden for this run only.

### AC-3: Trace records usage
- **Given** any AI call
- **When** the call completes
- **Then** `ai-trace.jsonl` includes `usage: { input, output, cumul }`
  on every line for providers that expose it.

### AC-4: Partial summary on halt
- **Given** the budget is exceeded mid-run
- **When** the framework halts
- **Then** a partial `RunSummary` is still written with
  `interrupted: true` and reason `budget-exceeded`.

### AC-5: Local providers warn
- **Given** Opencode/LM Studio (no token usage telemetry)
- **When** the chain uses such a provider
- **Then** the framework warns once per run that the budget is
  unenforceable but proceeds.
