# US-032 — Multi-browser matrix

## Story
As a **Test Lead**
I want to run the same suite on Chromium + Firefox + WebKit
So that I can report cross-browser compatibility.

## Acceptance criteria

### AC-1: Config-driven browsers
- **Given** `runner.browsers: ['chromium','firefox','webkit']`
- **When** a suite runs
- **Then** every scenario runs once per browser; results tag `browser`.

### AC-2: Compatibility matrix in report
- **Given** the report
- **When** I open the HTML
- **Then** a per-scenario table shows pass/fail per browser column.

### AC-3: Default unchanged
- **Given** no `runner.browsers` set
- **When** the suite runs
- **Then** behaviour is identical to PS-001 (chromium only).

### AC-4: Browser-specific defect
- **Given** a scenario fails only on `webkit`
- **When** validation runs
- **Then** the `suggestedDefect.summary` includes `(webkit only)`.
