# US-003 — Page Analyzer produces structured PageAnalysis

## Story
As an **automation tester**
I want to point the framework at a URL and get a structured PageAnalysis
So that downstream agents have grounding for generation and execution.

## Acceptance criteria

### AC-1: Valid URL returns PageAnalysis
- **Given** a reachable public URL
- **When** I run `npm run ai-test -- analyze --url https://example.com`
- **Then** the CLI writes
  `reports/evidence/{run-id}/page-analysis.json` whose content validates
  against the `PageAnalysis` Zod schema and contains:
  `title`, `url`, `viewport`, `capturedAt`, `screenshotPath`,
  `elements[]` (non-empty), `forms[]`, `navigation[]`.

### AC-2: Semantic locators
- **Given** the analyzer just analyzed a page with a `<button>Login</button>`
- **When** I inspect the produced `elements[]`
- **Then** at least one element has
  `locator.kind = "role"`, `locator.role = "button"`,
  `locator.name = "Login"`.

### AC-3: Required fields flagged
- **Given** the page has an input with `aria-required="true"`
- **When** the analyzer runs
- **Then** the corresponding element entry has `isRequired === true`.

### AC-4: Screenshot saved
- **Given** a successful analyze run
- **When** I inspect `reports/evidence/{run-id}/`
- **Then** a `page.png` file exists and is a valid PNG.

### AC-5: Unreachable URL fails clearly
- **Given** an unreachable URL `https://invalid.invalid.test`
- **When** the analyzer runs
- **Then** the CLI exits non-zero and stderr contains `Unreachable URL`
  with the underlying error reason.
