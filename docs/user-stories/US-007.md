# US-007 — JSON run report

## Story
As a **CI pipeline / test-management tool**
I want a machine-readable JSON report at a known location after every run
So that I can publish results to dashboards.

## Acceptance criteria

### AC-1: File created
- **Given** a completed run with id `R`
- **When** the run finishes
- **Then** `reports/json/R.json` exists.

### AC-2: Schema valid
- **Given** the produced JSON
- **When** validated against the `RunSummary` Zod schema
- **Then** it parses without errors and includes:
  `runId, startedAt, finishedAt, app, scenarios[], totals, environment`.

### AC-3: Sensitive values masked
- **Given** a scenario that filled a password field
- **When** the JSON is produced
- **Then** the captured value in the report shows `***` and the original
  value appears nowhere in the file.

### AC-4: Evidence paths relative
- **Given** any evidence reference in the JSON
- **When** I inspect it
- **Then** it is a path relative to the project root, not an absolute path.
