# US-020 — Multi-page run from SiteMap

## Story
As an **automation tester**
I want `ai-test run --site-map <id>` to iterate every page in the SiteMap
So that one command tests the whole application.

## Acceptance criteria

### AC-1: Run consumes SiteMap
- **Given** a SiteMap with N pages
- **When** I run `ai-test run --site-map {crawl-id} --mode explore`
- **Then** N PageAnalyses are produced and Test Design runs per page;
  totals reflect ≥N scenarios.

### AC-2: Per-page evidence isolated
- **Given** the multi-page run completes
- **When** I inspect `reports/evidence/{run-id}/`
- **Then** every page has its own sub-directory keyed by page hash with
  its own screenshots, trace, ai-trace.

### AC-3: storageState reused
- **Given** the run was launched after `ai-test auth login` produced a
  storageState
- **When** each page is opened
- **Then** authenticated content is visible (verified by snapshotting
  a known authenticated marker on at least one page).

### AC-4: Halt on first error is opt-in
- **Given** one page fails to load
- **When** the run continues with default `continueOnError: true`
- **Then** other pages still execute; failure is recorded per page;
  exit code 1.
