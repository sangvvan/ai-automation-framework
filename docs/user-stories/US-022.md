# US-022 — TestPlan generator

## Story
As a **Test Lead**
I want every run to produce a formal Test Plan document
So that I can show governance/audit reviewers a deliverable that
matches ISTQB language.

## Acceptance criteria

### AC-1: File produced
- **Given** any `ai-test run` against a SiteMap
- **When** the run completes
- **Then** `reports/test-plans/{run-id}.json` exists and validates
  against the `TestPlan` Zod schema with non-empty:
  `scope`, `testItems`, `levels`, `types`, `entryCriteria`,
  `exitCriteria`, `risks`, `resources`, `traceabilityMatrix`.

### AC-2: HTML tab
- **Given** the run completes
- **When** I open the HTML report
- **Then** a "Test Plan" tab renders all Plan sections with anchor
  links.

### AC-3: Traceability matrix populated
- **Given** REQ-009 (one REQ exists in `docs/requirements/`)
- **When** the Test Plan is generated
- **Then** the matrix lists at least one row per scenario keyed to
  its parent REQ and US (when known).

### AC-4: Deterministic narrative fallback
- **Given** no AI provider is configured
- **When** the Plan is generated
- **Then** narrative fields (approach, risks summary) fall back to a
  deterministic template; the Plan is still emitted.

### AC-5: TestPlan persisted in DB
- **Given** a successful run
- **When** I query `runs`
- **Then** `runs.test_plan_path` points to the JSON file.
