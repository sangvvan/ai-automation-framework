# US-016 — Crawl entry URL → SiteMap

## Story
As an **automation tester**
I want to run `ai-test crawl --url https://app.example.com`
So that I get a SiteMap covering every reachable page within the budget.

## Acceptance criteria

### AC-1: SiteMap produced
- **Given** a reachable public site with ≥5 internal links and default
  `maxPages: 25`, `maxDepth: 3`
- **When** I run `npm run ai-test -- crawl --url https://x`
- **Then** the CLI writes `reports/sitemaps/{crawl-id}.json` validating
  the `SiteMap` Zod schema with ≥5 `pages[]` entries and an `exitReason`.

### AC-2: Route patterns collapsed
- **Given** the site has `/posts/1`, `/posts/2`, `/posts/3`
- **When** crawl completes
- **Then** the SiteMap contains exactly one entry with
  `routePattern: "/posts/:id"` and `sampleUrl` set to one observed URL.

### AC-3: Same-origin by default
- **Given** the site links to an external host
- **When** crawl runs without `--include-subdomains`
- **Then** the external host is skipped and recorded under
  `skipped: [{ url, reason: "out-of-scope" }]`.

### AC-4: Budget honoured
- **Given** `maxPages: 10`
- **When** crawl finds 50 candidates
- **Then** exactly 10 are fetched and `exitReason: "maxPagesReached"`.
