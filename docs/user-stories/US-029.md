# US-029 — Extended assertions

## Story
As an **automation tester**
I want richer expected-result fields (negation, visibility, attributes,
child count)
So that I can express realistic test oracles without writing custom code.

## Acceptance criteria

### AC-1: Schema extended
- **Given** the codebase
- **When** I import `ExpectedResult`
- **Then** it has optional fields `urlNotContains`, `textNotContains`,
  `visibleLocators[]`, `notVisibleLocators[]`, `attribute`,
  `childCount`.

### AC-2: Validator dispatches each
- **Given** an `ExpectedResult` with multiple new fields set
- **When** validation runs
- **Then** `ValidationResult.checks[]` has one entry per supplied field.

### AC-3: Each has accept + reject test
- **Given** the validator codebase
- **When** Vitest runs
- **Then** each new field has at least one accept and one reject test.

### AC-4: Backwards compat
- **Given** an old `ExpectedResult` with only `url` + `text`
- **When** parsed
- **Then** behaviour is unchanged from PS-001.
