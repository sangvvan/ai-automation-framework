# US-040 — Locale switching

## Story
As a **tester of a multilingual site**
I want to run the same suite across multiple locales
So that I catch locale-specific layout or validation issues.

## Acceptance criteria

### AC-1: --locales flag
- **Given** `--locales en,vi,ja`
- **When** the suite runs
- **Then** each scenario runs once per locale; results are tagged
  `locale`.

### AC-2: Accept-Language header set
- **Given** `--locales vi`
- **When** the runner opens a page
- **Then** the request's `Accept-Language` header includes `vi`.

### AC-3: `<html lang>` verified
- **Given** a multilingual page that swaps `lang` attribute
- **When** the scenario runs in a non-default locale
- **Then** a validator check `langAttribute` records pass/fail
  against the expected locale.

### AC-4: Report groups by locale
- **Given** the HTML report
- **When** I open it
- **Then** a "Locale" filter chip and per-locale roll-ups appear.
