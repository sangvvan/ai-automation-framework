# US-031 — Test level field

## Story
As a **Test Lead**
I want every run to declare its test level (system / acceptance / …)
So that reports do not silently mislabel the scope.

## Acceptance criteria

### AC-1: RunSummary field
- **Given** the codebase
- **When** I import `RunSummary`
- **Then** it has `testLevel: 'system'|'acceptance'|'integration'|
  'component'|'unit'` defaulting to `'system'`.

### AC-2: TestPlan declares level
- **Given** a generated TestPlan
- **When** I parse the JSON
- **Then** `levels[]` is non-empty and includes the run's testLevel.

### AC-3: HTML header shows level
- **Given** the HTML report
- **When** I open it
- **Then** the header pill shows the test level next to the mode badge.

### AC-4: CLI flag
- **Given** the run subcommand
- **When** I pass `--test-level acceptance`
- **Then** RunSummary records that level and the report reflects it.
