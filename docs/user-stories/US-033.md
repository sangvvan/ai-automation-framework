# US-033 — Self-healing locators

## Story
As an **automation tester**
I want a failing locator to attempt to heal itself
So that minor UI changes do not break the suite immediately.

## Acceptance criteria

### AC-1: Opt-in via config
- **Given** `runner.selfHeal: true`
- **When** a primary locator fails to resolve
- **Then** the framework tries candidate fuzzy matches deterministically.

### AC-2: Audit recorded
- **Given** a successful heal
- **When** the run completes
- **Then** `StepResult.healEvent` records `from`/`to` locators and a
  `confidence` score.

### AC-3: Healed scenarios flagged
- **Given** at least one heal
- **When** I open the HTML report
- **Then** the scenario shows a "healed" badge with the heal count.

### AC-4: Same-kind only
- **Given** a `role` locator fails
- **When** healing runs
- **Then** only other `role` candidates are tried, not `text`/`testId`.

### AC-5: Off by default
- **Given** no `selfHeal` config
- **When** locators fail
- **Then** behaviour is identical to PS-001 (no healing).
