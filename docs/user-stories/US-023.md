# US-023 — Test Suite layer

## Story
As a **Test Lead**
I want generated scenarios grouped into Test Suites by feature/page
So that I can review, re-run, and report at the suite level.

## Acceptance criteria

### AC-1: Suite created per page
- **Given** a multi-page run
- **When** Test Design completes
- **Then** the DB has one `test_suites` row per analysed page (named
  after the page title) and every scenario carries `suite_id`.

### AC-2: Ad-hoc suite for orphans
- **Given** a scenario not attached to any page (e.g. a manually
  approved YAML)
- **When** it runs
- **Then** it attaches to a singleton `Ad-hoc` suite per run.

### AC-3: Suite re-execution
- **Given** an approved suite
- **When** I run `ai-test run --suite-id <id>`
- **Then** only scenarios in that suite execute; report is suite-scoped.

### AC-4: Setup / teardown hooks
- **Given** a suite with `setup_hook` and `teardown_hook` script paths
- **When** the suite runs
- **Then** setup runs once before scenarios, teardown runs once after,
  both timed and surfaced in the report.

### AC-5: Regression tag
- **Given** a suite tagged `regression`
- **When** I run `ai-test run --regression`
- **Then** only suites with that tag execute.
