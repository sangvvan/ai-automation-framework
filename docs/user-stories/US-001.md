# US-001 — `ai-test` CLI is installable and prints help

## Story
As an **automation tester**
I want to install the framework and run `ai-test --help`
So that I can discover the available commands without reading source code.

## Acceptance criteria

### AC-1: Help lists every subcommand
- **Given** the framework is installed (npm script entrypoint wired)
- **When** I run `npm run ai-test -- --help`
- **Then** the output lists at least: `run`, `analyze`, `report`, `config`,
  each with a one-line description and an example.

### AC-2: Subcommand help works
- **Given** the CLI is installed
- **When** I run `npm run ai-test -- run --help`
- **Then** the output lists `--url`, `--test-case`, `--mode`, `--output-dir`
  with type and default value.

### AC-3: Unknown command exits non-zero
- **Given** the CLI is installed
- **When** I run `npm run ai-test -- nonsense`
- **Then** the process exits with code `1` and a "Unknown command" message
  is written to stderr.
