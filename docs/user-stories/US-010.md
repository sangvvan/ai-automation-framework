# US-010 — Auth: register, log in, log out

## Story
As a **reviewer**
I want to register with email + password and sign in to the Review web UI
So that I can act on generated scenarios.

## Acceptance criteria

### AC-1: Registration creates user
- **Given** the registration form
- **When** I submit valid email + password (≥10 chars, ≥1 digit) + name
- **Then** a user row is created with bcrypt-hashed password, role
  defaults to `tester`, and I am redirected to `/runs`.

### AC-2: Login sets cookie session
- **Given** an existing user
- **When** I submit correct credentials
- **Then** a signed session cookie is set and I am redirected to `/runs`.

### AC-3: Bad credentials → 422 + fieldErrors
- **Given** the login form
- **When** I submit a wrong password
- **Then** the response is HTTP 422 with `fieldErrors.password` populated
  and no user enumeration leak (same message for missing email).

### AC-4: Logout clears session
- **Given** I am signed in
- **When** I POST to `/auth/logout`
- **Then** the session cookie is cleared and `/runs` redirects to login.

### AC-5: Protected routes require auth
- **Given** I am not signed in
- **When** I visit `/runs` or any `/runs/*` route
- **Then** I am redirected to `/auth/login?next=...`.
