# US-028 — JUnit XML reporter

## Story
As a **CI pipeline** (Jenkins / GitLab / Azure)
I want a JUnit-XML compatible report at a known path
So that I can render test results in the native CI UI.

## Acceptance criteria

### AC-1: File emitted
- **Given** any completed run
- **When** the reporter step finishes
- **Then** `reports/junit/{run-id}.xml` exists and conforms to the
  Jenkins JUnit XSD.

### AC-2: Suite mapping
- **Given** a run with N Test Suites
- **When** I parse the XML
- **Then** the file contains a root `<testsuites>` with N
  `<testsuite>` children; each `<testsuite>` has a matching
  `<testcase>` count.

### AC-3: Counts match
- **Given** the run totals show `passed: 7, failed: 2, skipped: 1`
- **When** I parse the XML attributes
- **Then** the root `<testsuites>` shows `tests=10 failures=2
  errors=0 skipped=1`.

### AC-4: Failure detail
- **Given** a failed `<testcase>`
- **When** I inspect it
- **Then** the `<failure>` element has a non-empty `message` attr
  and the body contains the suggestedDefect summary.

### AC-5: System-out evidence
- **Given** a `<testcase>`
- **When** I read its `<system-out>` block
- **Then** it lists the relative paths to screenshot and trace.
