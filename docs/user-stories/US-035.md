# US-035 — Defect persistence

## Story
As a **reviewer**
I want a defects page that lists every AI-suggested defect from past runs
So that I can triage instead of re-reading reports.

## Acceptance criteria

### AC-1: Defect row inserted
- **Given** a scenario fails with `suggestedDefect`
- **When** the validator runs
- **Then** a `defects` row exists with status `open`.

### AC-2: Defects page lists them
- **Given** I am authenticated and visit `/runs/:id/defects`
- **When** the page loads
- **Then** I see a table of defects with filters by status/severity
  and a "Copy as issue body" button per row.

### AC-3: Status transitions
- **Given** I am `tester` or `test-lead`
- **When** I change a defect status
- **Then** the DB row updates and an `audit_log` row is written.

### AC-4: External ref editable
- **Given** I paste a Jira/GitHub URL into the external_ref field
- **When** I save
- **Then** the URL is persisted and rendered as a link in the list.

### AC-5: Defects link back to scenario + run
- **Given** any defect
- **When** I click on it
- **Then** I land on the scenario detail with the relevant evidence
  highlighted.
