# US-004 — YAML test-case parser

## Story
As an **automation tester**
I want to write a YAML test case and have the framework parse + validate it
So that I can run it without writing automation code.

## Acceptance criteria

### AC-1: Valid YAML parses
- **Given** a YAML file conforming to PS §6.2 schema
- **When** `parseTestCaseFile(path)` runs
- **Then** it returns `ExecutableScenario[]` (length ≥ 1) and no errors.

### AC-2: Field-level validation
- **Given** a YAML missing `steps`
- **When** `parseTestCaseFile(path)` runs
- **Then** it throws `TestCaseValidationError` whose `issues[]` contains
  `{ path: ["test_cases", 0, "steps"], message: "Required" }`.

### AC-3: Step → keyword mapping
- **Given** a step `"Click Login"` and a PageAnalysis containing a button
  with accessible name "Login"
- **When** the mapper runs
- **Then** the resulting `Action` is
  `{ keyword: "click", target: { role: "button", name: "Login" } }`.

### AC-4: Unmappable step produces warning, not silent skip
- **Given** a step `"Click Foo"` and no matching element
- **When** the mapper runs
- **Then** the resulting `ExecutableScenario.warnings[]` includes
  `{ stepIndex, reason: "no matching element" }` and the step is marked
  `unresolved`.
