# US-005 — Test Execution Engine runs scenarios end-to-end

## Story
As an **automation tester**
I want the framework to execute a parsed scenario via Playwright
So that I get a deterministic pass/fail result with evidence.

## Acceptance criteria

### AC-1: Happy path passes
- **Given** an ExecutableScenario whose steps and expectations all resolve
- **When** the runner executes it against the target app
- **Then** the produced `ScenarioResult.status === "passed"` and every
  step in `steps[].status === "passed"`.

### AC-2: Failure halts scenario and captures evidence
- **Given** a step whose locator does not resolve within `stepTimeoutMs`
- **When** the runner executes
- **Then** `ScenarioResult.status === "failed"`, the failing step has
  `status: "failed"` with `reason: "timeout"`, and the evidence directory
  contains `failure.png` and `trace.zip`.

### AC-3: Keywords-only API
- **Given** the codebase
- **When** lint runs
- **Then** no file under `app/lib/runner/` calls `page.locator` or `page.$`
  outside the keyword library.

### AC-4: Console errors captured
- **Given** a scenario that triggers a JS console error
- **When** the runner finishes that scenario
- **Then** `ScenarioResult.consoleMessages[]` contains an entry with
  `level: "error"` and the original message.

### AC-5: Headless by default
- **Given** no `--headed` flag
- **When** the runner starts
- **Then** Playwright launches in headless mode.
