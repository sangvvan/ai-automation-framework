# US-015 — End-to-end MVP smoke story

## Story
As an **automation tester**
I want to run `ai-test run --url … --mode explore` and open the resulting
HTML report
So that I can validate the framework end-to-end in one command.

## Acceptance criteria

### AC-1: Single-command run succeeds
- **Given** a reachable URL and a valid config
- **When** I run `npm run ai-test -- run --url https://example.com --mode explore`
- **Then** the CLI exits 0 (or non-zero only when scenarios genuinely
  failed), prints a summary line, and prints the path to the HTML report.

### AC-2: Artifacts present
- **Given** the run completed
- **When** I inspect the filesystem
- **Then** `reports/json/{run-id}.json`,
  `reports/html/{run-id}/index.html`, and at least one screenshot under
  `reports/evidence/{run-id}/` exist.

### AC-3: Database row
- **Given** the run completed
- **When** I query the `runs` table
- **Then** a row exists with the run id, mode `explore`, totals, and
  links to the report paths.
