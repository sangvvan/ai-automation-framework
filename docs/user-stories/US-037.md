# US-037 — Element coverage heatmap

## Story
As a **Test Lead**
I want to see which UI elements were exercised and which were not
So that I can spot gaps in the AI's exploration.

## Acceptance criteria

### AC-1: Touched elements tracked
- **Given** a scenario uses `click`/`fill`/`select`/`wait_for`
- **When** the run completes
- **Then** each touched `PageElement.id` is recorded on the
  `RunSummary.elementCoverage` map.

### AC-2: Per-page coverage %
- **Given** a page with 20 visible interactive elements and 8 touched
- **When** I open the HTML report
- **Then** the page row shows `40% (8/20)` with a colour gradient.

### AC-3: Untested elements listed
- **Given** the coverage panel
- **When** I expand a page
- **Then** untested elements are listed with their accessibleName +
  locator so a reviewer can decide to add a scenario.

### AC-4: Coverage JSON
- **Given** any run
- **When** I inspect `reports/coverage/{run-id}.json`
- **Then** it validates the `ElementCoverage` Zod schema.
