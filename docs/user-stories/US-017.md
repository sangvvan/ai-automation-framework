# US-017 — Polite crawler (robots + rate limit)

## Story
As a **framework operator**
I want the crawler to respect robots.txt and a per-host QPS cap
So that the framework is a polite citizen on the open web.

## Acceptance criteria

### AC-1: robots.txt enforced by default
- **Given** the site's robots.txt disallows `/admin`
- **When** crawl runs
- **Then** `/admin` is **not** fetched; the SiteMap records
  `skipped: [{ url, reason: "robots-disallow" }]`.

### AC-2: --ignore-robots opt-out
- **Given** the same site
- **When** I pass `--ignore-robots`
- **Then** `/admin` **is** fetched and the audit log records the
  override with the actor + timestamp.

### AC-3: Rate limit honoured
- **Given** `perHostQps: 2`
- **When** crawl fetches 10 pages from the same host
- **Then** total elapsed time ≥ 5s and no two requests are <500ms
  apart (verified via injected clock in tests).

### AC-4: User-Agent set
- **Given** any crawl
- **When** I inspect a captured request
- **Then** the `User-Agent` header contains `ai-test/<version>`.
