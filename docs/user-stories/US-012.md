# US-012 — Approve scenario → write to tests/approved/

## Story
As a **tester reviewer**
I want to approve a generated scenario
So that it becomes part of the approved test corpus that can later be promoted.

## Acceptance criteria

### AC-1: Approve action
- **Given** a scenario with status `pending_review`
- **When** I click "Approve" on the scenario detail page
- **Then** the DB row updates to `status: approved`, with reviewer id +
  timestamp, and a confirmation toast appears.

### AC-2: YAML file written
- **Given** the approval succeeded
- **When** I inspect the filesystem
- **Then** a file
  `tests/approved/{feature-slug}/{scenario-id}.yaml`
  exists with the canonical scenario shape and includes an
  `approvedBy`/`approvedAt` frontmatter.

### AC-3: Auth required
- **Given** I am unauthenticated
- **When** I POST `/runs/R/scenarios/S/approve`
- **Then** the response is 302 to `/auth/login` and no DB or file changes occur.

### AC-4: Audit log
- **Given** an approval succeeds
- **When** I query `audit_log`
- **Then** a row exists with `actor`, `action: "approve"`,
  `entity: "scenario:{id}"`, `at`, and the scenario snapshot JSON.
