# US-036 — GitHub PR comment

## Story
As a **developer opening a PR**
I want to see the framework's run summary as a PR comment
So that I do not need to dig into CI artefacts.

## Acceptance criteria

### AC-1: Comment posted
- **Given** `GITHUB_TOKEN`, `GITHUB_REPOSITORY`, `GITHUB_PR_NUMBER` set
- **When** the reporter runs
- **Then** a comment is posted to the PR containing totals, top 5
  failures, and links to JUnit + HTML artefacts.

### AC-2: Idempotent update
- **Given** a second run on the same PR
- **When** the reporter runs
- **Then** the existing comment (matched by hidden marker) is updated
  in place; no duplicate.

### AC-3: Skipped without env
- **Given** missing `GITHUB_TOKEN`
- **When** the reporter runs
- **Then** the step is skipped silently; exit code unchanged.

### AC-4: Body cap
- **Given** a run with 200 failures
- **When** the comment is built
- **Then** the body is ≤ 64 KB; overflow lists "and N more — see HTML
  report".

### AC-5: Token never logged
- **Given** any run
- **When** I `grep $GITHUB_TOKEN` reports + logs
- **Then** zero matches.
