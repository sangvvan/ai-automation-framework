# US-027 — Web Vitals capture

## Story
As a **developer reviewing test results**
I want LCP / CLS / INP / TTFB recorded per page navigation
So that I see performance regressions in the same report as functional ones.

## Acceptance criteria

### AC-1: Metrics populated
- **Given** any scenario that opens at least one page
- **When** the run finishes
- **Then** `ScenarioResult.webVitals` contains numeric `lcpMs`,
  `cls`, `inpMs`, `ttfbMs` (any missing metric defaults to null with
  a `notMeasurable: true` flag).

### AC-2: Thresholds configurable
- **Given** `report.performance.lcpMs: 2500`
- **When** a page measures LCP = 3000ms
- **Then** ValidationResult adds a `performance` check with status
  `warn` (3000 < 2×2500); at LCP > 5000 status is `failed`.

### AC-3: Report panel
- **Given** the HTML report
- **When** I expand a scenario
- **Then** I see a small "Vitals" table with values + colour-coded
  thresholds.

### AC-4: web-vitals served locally
- **Given** any run
- **When** I inspect the network log
- **Then** no requests go to a CDN for the web-vitals script (served
  inline / bundled).
