# US-034 — Screenshot diff baselining

## Story
As a **tester**
I want a `verify_screenshot` keyword that diffs against a baseline
So that I detect visual regressions automatically.

## Acceptance criteria

### AC-1: First run captures baseline
- **Given** no baseline exists for the named shot
- **When** the scenario runs
- **Then** `reports/baselines/{suite}/{scenario}/{name}.png` is written
  and the step is marked `passed` with `firstBaseline: true`.

### AC-2: Second run diffs
- **Given** the baseline exists and the UI is unchanged
- **When** the scenario runs again
- **Then** the step passes; reported delta < threshold.

### AC-3: Drift triggers fail
- **Given** the UI shifted > threshold
- **When** the scenario runs
- **Then** the step fails with reason `screenshot-diff` and writes a
  side-by-side diff image under the run evidence dir.

### AC-4: Baseline acceptance
- **Given** an intentional UI change
- **When** I run `ai-test baselines accept --run-id <id>`
- **Then** matching baselines are overwritten; audit log records each
  replacement.

### AC-5: Sensitive fields blurred
- **Given** a page with password fields
- **When** the baseline is captured
- **Then** those fields are blurred before saving.
