# US-014 — Promote to regression (test-lead only)

## Story
As a **Test Lead**
I want to promote an approved scenario into the regression suite
So that future regression runs include it.

## Acceptance criteria

### AC-1: Role gate
- **Given** I am signed in as `tester` (not `test-lead`)
- **When** I open the scenario detail page
- **Then** the "Promote to regression" button is disabled with a tooltip
  "Test Lead role required".

### AC-2: Promotion writes file
- **Given** I am `test-lead` and the scenario is `approved`
- **When** I click "Promote to regression"
- **Then** a file `tests/regression/{feature-slug}/{scenario-id}.yaml`
  is created and the DB row is updated to `inRegression: true`.

### AC-3: Server enforces role
- **Given** a `tester` POSTs `/runs/R/scenarios/S/promote`
- **When** the action runs
- **Then** the response is 403 and no changes happen.

### AC-4: Audit log
- **Given** a promotion succeeds
- **When** I query `audit_log`
- **Then** a row exists with `action: "promote"`.
