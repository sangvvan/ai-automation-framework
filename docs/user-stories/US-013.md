# US-013 — Reject scenario with reason

## Story
As a **tester reviewer**
I want to reject a scenario with a reason
So that low-value or incorrect scenarios are filtered out and the rationale is preserved.

## Acceptance criteria

### AC-1: Reason required
- **Given** the Reject dialog
- **When** I submit with reason length < 10
- **Then** the form returns 422 with `fieldErrors.reason`.

### AC-2: Rejection persisted
- **Given** a scenario in `pending_review`
- **When** I submit Reject with reason ≥ 10 chars
- **Then** the DB row updates to `status: rejected`, reason saved,
  reviewer id + timestamp recorded.

### AC-3: Hidden from default regression
- **Given** a rejected scenario
- **When** the next regression run lists scenarios
- **Then** the rejected scenario is excluded by default and only
  re-included with `--include-rejected`.

### AC-4: Audit log
- **Given** a rejection succeeds
- **When** I query `audit_log`
- **Then** a row exists with `action: "reject"`, reason stored.
