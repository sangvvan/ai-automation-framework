# US-011 — Runs list and pending scenarios

## Story
As a **reviewer**
I want to see all recent runs and the scenarios pending my review
So that I can decide what to work on.

## Acceptance criteria

### AC-1: Runs index lists recent runs
- **Given** I am signed in as `tester` or `test-lead`
- **When** I open `/runs`
- **Then** a table lists the last 50 runs sorted by `startedAt desc`
  with columns: id, app, mode, totals, status, started.

### AC-2: Run detail page
- **Given** a run with id `R`
- **When** I open `/runs/R`
- **Then** I see summary KPIs and a list of scenarios with their review
  status (`pending_review`, `approved`, `rejected`).

### AC-3: Pending filter
- **Given** the run detail page
- **When** I click the "Pending" filter
- **Then** only scenarios with status `pending_review` are shown.

### AC-4: Empty state
- **Given** there are no runs yet
- **When** I open `/runs`
- **Then** an empty-state component invites me to run the CLI with the
  exact command to copy.
