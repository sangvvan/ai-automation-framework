# US-002 — Configuration loader (YAML + env, Zod-validated)

## Story
As a **framework operator**
I want the framework to read `configs/*.yaml` and merge it with environment
variables at startup
So that I get a single validated configuration object and fail-fast on bad config.

## Acceptance criteria

### AC-1: Valid config loads
- **Given** `configs/framework.config.yaml`, `configs/test-env.yaml`,
  `configs/ai-provider.yaml` exist with valid content
- **When** the framework starts (CLI or web)
- **Then** `loadConfig()` returns a Zod-typed `FrameworkConfig` and no
  errors are thrown.

### AC-2: Missing required key fails fast
- **Given** `ai-provider.yaml` is missing `defaultProvider`
- **When** `loadConfig()` runs
- **Then** it throws `ConfigError` whose message names the file and path
  (e.g. `ai-provider.yaml: defaultProvider is required`).

### AC-3: Env overrides YAML
- **Given** `framework.config.yaml` sets `baseUrl: "http://a"` and
  env `AI_TEST_BASE_URL=http://b` is set
- **When** `loadConfig()` runs
- **Then** the loaded config exposes `baseUrl === "http://b"`.

### AC-4: Only config module reads `process.env`
- **Given** the codebase
- **When** lint runs
- **Then** `process.env` is only referenced inside `app/lib/config.ts`.
