# US-006 — Result validation produces a pass/fail decision

## Story
As a **tester**
I want the framework to validate scenario results against expectations
So that I get a precise reason for any failure plus a defect summary.

## Acceptance criteria

### AC-1: Expected URL matches actual → pass
- **Given** a ScenarioResult whose final URL matches the scenario's
  `expectedResult.url`
- **When** validation runs
- **Then** `ValidationResult.status === "passed"` and `checks[]` contains
  a URL check with `status: "passed"`.

### AC-2: Expected text missing → fail with reason
- **Given** an `expectedResult.text` not present in the captured page text
- **When** validation runs
- **Then** `ValidationResult.status === "failed"` and `failureReason`
  contains both the expected and the actual snippet (truncated to 200ch).

### AC-3: Console errors → WARN on pass
- **Given** a scenario that passes functional asserts but recorded a
  console error
- **When** validation runs
- **Then** `ValidationResult.status === "passed"` and `checks[]` contains
  a console-error check with `severity: "warn"`.

### AC-4: Suggested defect on failure
- **Given** any `ValidationResult.status === "failed"`
- **When** validation completes
- **Then** the result includes `suggestedDefect` with `summary`,
  `stepsToReproduce[]`, `evidenceLinks[]`, and `severity`.
