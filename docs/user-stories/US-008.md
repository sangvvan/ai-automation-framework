# US-008 — HTML run report

## Story
As a **tester or Test Lead**
I want a human-readable HTML report I can open in a browser or share by zip
So that I do not need to read the JSON manually.

## Acceptance criteria

### AC-1: Index page opens
- **Given** a completed run
- **When** I open `reports/html/R/index.html`
- **Then** the page loads with no JS console errors and renders
  KPI cards: total / passed / failed / skipped.

### AC-2: Failure detail
- **Given** at least one failed scenario in the run
- **When** I open the report
- **Then** the failed scenario row expands to show step-by-step results,
  failure reason, and a link to the screenshot.

### AC-3: Self-contained
- **Given** the report directory
- **When** I copy `reports/html/R/` to another machine
- **Then** the report renders correctly offline (no CDN dependencies).

### AC-4: Regression diff section
- **Given** a previous RunSummary exists for the same suite tag
- **When** I open the new report
- **Then** a "Regression diff" section lists new failures and
  newly-passing tests vs the previous run.
