# US-025 — Technique-driven generation

## Story
As an **automation tester**
I want to ask the AI for scenarios by technique
So that I can deliberately target boundary or decision-table coverage.

## Acceptance criteria

### AC-1: --techniques flag
- **Given** `--techniques boundary-value,decision-table`
- **When** explore mode runs
- **Then** generated scenarios only carry those two `designTechnique`
  values.

### AC-2: Default covers all
- **Given** no `--techniques` flag
- **When** generation runs against a page with inputs
- **Then** the output contains at least one scenario per:
  `equivalence-partition`, `boundary-value`, `error-guessing`.

### AC-3: Per-technique prompt addendum
- **Given** the agent generates a decision-table scenario
- **When** I read the AI trace
- **Then** the system prompt addendum describing decision tables is
  appended to the base prompt for that call.

### AC-4: Coverage panel in report
- **Given** any explore run
- **When** I open the HTML report
- **Then** a "Technique coverage" panel lists count + pass-rate per
  technique observed in the run.
