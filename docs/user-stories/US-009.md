# US-009 — AI scenario generation (explore mode)

## Story
As an **automation tester with no test cases**
I want the framework to generate candidate scenarios from a URL alone
So that I can review them and seed a regression suite.

## Acceptance criteria

### AC-1: Categories covered
- **Given** a PageAnalysis with at least one form
- **When** I run `ai-test run --url … --mode explore`
- **Then** the produced `GeneratedScenario[]` contains at least one
  scenario each of: `positive`, `negative`, `required-field`,
  `boundary`, `navigation`.

### AC-2: Strict JSON schema
- **Given** generation completes
- **When** each scenario is validated against the `GeneratedScenario`
  Zod schema
- **Then** all scenarios validate (model output is rejected and re-asked
  if it does not).

### AC-3: Steps reference real elements
- **Given** a generated scenario
- **When** I inspect each step
- **Then** every locator referenced in the step exists in the PageAnalysis
  passed in.

### AC-4: maxScenarios respected
- **Given** `generation.maxScenarios = 10`
- **When** generation runs
- **Then** `GeneratedScenario.length <= 10`.

### AC-5: Provider fallback
- **Given** the primary provider returns 5xx
- **When** generation runs
- **Then** the framework retries on the configured fallback provider per
  CLAUDE.md routing chain, and the audit log records both attempts.

### AC-6: No PII harvesting
- **Given** a page contains user names in a table
- **When** generation produces a scenario that fills a name field
- **Then** the value is synthetic (does not match any name observed in
  the PageAnalysis text).
