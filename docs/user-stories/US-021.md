# US-021 — CLI `crawl` + `auth` subcommands

## Story
As an **automation tester**
I want first-class CLI subcommands for crawl and auth flows
So that I can compose them in scripts and CI.

## Acceptance criteria

### AC-1: `ai-test crawl --help`
- Lists `--url` (required), `--max-pages`, `--max-depth`,
  `--max-concurrency`, `--per-host-qps`, `--include-subdomains`,
  `--ignore-robots`, `--auth-recipe`, `--output-dir`.

### AC-2: `ai-test auth detect --help`
- Lists `--url`, `--output` (default `inputs/auth/{host}.draft.yaml`).

### AC-3: `ai-test auth login --help`
- Lists `--recipe` (required), `--output-storage` (default per-run).

### AC-4: `ai-test run --site-map` works
- Adds `--site-map <crawl-id>` flag to the existing `run` subcommand;
  mutually exclusive with `--url`.

### AC-5: Exit codes consistent
- All new subcommands honour the same exit-code convention as PS-001:
  0 ok, 1 assertion fail, 2 orchestration error, 3 auth error.
