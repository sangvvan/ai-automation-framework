# REQ-008 — CLI, Configuration, and Auth Platform

## Purpose
Cross-cutting platform requirement: provide the `ai-test` CLI, the YAML
configuration loader, the AI-provider configuration with fallback chains,
and the authentication / session layer used by the Review web UI
(REQ-007). Without this REQ the other REQs have no runnable surface.

## Scope
- **CLI** (`ai-test` binary):
  - `ai-test run --url <URL> [--test-case <FILE>] --mode (testcase|explore)`
  - `ai-test analyze --url <URL>` (Page Analyzer only)
  - `ai-test report --run-id <ID>` (open the HTML report path)
  - `ai-test config show` (effective merged config)
- **Config**: YAML files under `configs/` (`framework.config.yaml`,
  `test-env.yaml`, `ai-provider.yaml`) merged with environment
  variables. Validated by Zod at startup.
- **Provider routing**: Honour fallback chains from CLAUDE.md
  (`claude → codex`, `codex → opencode → opencode-ollama`).
- **Auth (Web)**: Email + password registration, cookie session, password
  hashed with bcrypt. Roles: `viewer`, `tester`, `test-lead`.
- **Session helper**: `requireUser(request, role?)` used by all protected
  loaders/actions in REQ-007.

## Out of scope
- SSO / OAuth providers (future).
- Multi-tenancy.
- Granular per-resource ACL (role-only in MVP).

## Acceptance criteria
- AC-008-1: `ai-test --help` lists all commands; each subcommand has its
  own `--help`.
- AC-008-2: `ai-test run --url https://… --mode explore` exits 0 on a
  successful run and non-zero on any failed scenario.
- AC-008-3: Config validation rejects an unknown AI provider name with
  a precise message and path.
- AC-008-4: A tester can register, log in, and access `/runs` only when
  authenticated; unauthenticated requests redirect to `/auth/login`.
- AC-008-5: Passwords are stored bcrypt-hashed; sessions are signed with
  `SESSION_SECRET` and expire after 7 days.
- AC-008-6: A `test-lead` user can access the Promote-to-Regression
  control; a `tester` user receives 403 on the same action.

## Constraints
- Only `app/lib/config.ts` reads `process.env`.
- Passwords hashed with bcrypt (cost ≥10); never logged.
- No CLI command writes outside the project's `reports/`, `tests/`, and
  config-allowed paths.

## Inputs
- CLI args and YAML configs
- Environment variables (`DATABASE_URL`, `SESSION_SECRET`, provider keys)

## Outputs
- CLI exit codes + console output
- Auth cookie + DB rows: `users`, `sessions`
