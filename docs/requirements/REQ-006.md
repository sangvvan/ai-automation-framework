# REQ-006 — Reporting

## Purpose
Aggregate per-scenario results and validation outcomes into a structured
report suitable for testers, developers, and Test Leads. Provide both a
machine-readable JSON report and a human-readable HTML report, with all
evidence (screenshots, traces, logs) reachable by relative links.

## Scope
- Generate `reports/json/{run-id}.json` capturing the full RunSummary.
- Generate `reports/html/{run-id}/index.html` with:
  summary KPIs (total/passed/failed/skipped), per-page coverage, failure
  detail with screenshots, links to Playwright trace, AI observations,
  suggested next actions.
- Group results by page / feature / user journey.
- Highlight new failures vs previous run (regression diff) when a previous
  RunSummary exists for the same suite.
- Web UI dashboard exposes the latest N runs (REQ-007 wires this up).

## Out of scope
- Live streaming of in-progress runs (future REQ).
- Multi-tenant cross-project dashboards.

## Acceptance criteria
- AC-006-1: After a run, `reports/json/{run-id}.json` exists and conforms
  to the `RunSummary` Zod schema.
- AC-006-2: After a run, `reports/html/{run-id}/index.html` opens in a
  browser and renders summary KPIs + failure detail without console errors.
- AC-006-3: Every failure entry links to its screenshot and trace file
  via a relative path; clicking the link works locally.
- AC-006-4: Sensitive values (password fields, configured PII keys) are
  masked in both JSON and HTML reports.
- AC-006-5: A regression diff section appears when a prior RunSummary
  for the same suite exists, listing new failures and newly-passing tests.

## Constraints
- HTML report is self-contained (CSS inlined or bundled) so it can be
  archived and viewed offline.
- No outbound network calls from the rendered HTML.

## Inputs
- `RunValidation` (from REQ-005)
- Previous `RunSummary` for the same suite (optional, for diff)

## Outputs
- `reports/json/{run-id}.json`
- `reports/html/{run-id}/index.html`
- Coverage summary by page / feature / user journey
