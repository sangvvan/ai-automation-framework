# REQ-003 — AI Scenario Generation

## Purpose
When the tester supplies only a URL (explore mode), the Test Design Agent
takes the PageAnalysis from REQ-001 and produces a structured set of
test scenarios covering positive, negative, boundary, validation,
navigation, UI consistency, error handling, basic accessibility, and
basic security (invalid-input) checks.

## Scope
- Call the configured AI provider (Claude default, fallback per CLAUDE.md
  routing rules) with PageAnalysis as grounding.
- Constrain the model to a Zod-validated JSON schema for scenarios.
- Tag each scenario with `type`, `priority` (P1/P2/P3), and `risk`.
- Cap scenarios per page (configurable, default 25) to avoid runaway costs.
- Allow a tester to pass optional acceptance criteria / business rules
  that bias generation.

## Out of scope
- Multi-page user-journey generation (future REQ).
- Self-healing of broken locators (future REQ).
- Fully autonomous bug filing (out of MVP, per PS §14.2).

## Acceptance criteria
- AC-003-1: Given a PageAnalysis with ≥1 form, the generator returns at
  least one scenario per category: positive, negative, required-field,
  boundary, navigation.
- AC-003-2: Every generated scenario validates against the Zod schema
  (id, title, type, priority, steps[], expected_result).
- AC-003-3: Each scenario step references an element that exists in the
  PageAnalysis (locator strategy + value or accessible name).
- AC-003-4: Generation respects `maxScenarios` and never exceeds it.
- AC-003-5: When the AI provider fails (timeout, 5xx, schema mismatch),
  the framework falls back to the configured secondary provider per
  CLAUDE.md `CLAUDE_FALLBACK`/`CODEX_FALLBACK` chain.
- AC-003-6: Sensitive fields (passwords, PII) are seeded with synthetic
  test data, never with values harvested from the page.

## Constraints
- Use structured outputs (JSON schema) — never trust free-form model text.
- No outbound AI calls when `AI_PROVIDER=local-only` mode is set.
- All provider responses are persisted to
  `reports/evidence/{run-id}/ai-trace.json` for audit.

## Inputs
- `PageAnalysis` (REQ-001)
- Optional `acceptanceCriteria: string[]`
- Optional `businessRules: string[]`
- Generation config: `maxScenarios`, `categories[]`, `priority`

## Outputs
- `GeneratedScenario[]` ready for review (REQ-007) or direct execution
