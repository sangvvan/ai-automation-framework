# REQ-009 â€” Web Crawler & SiteMap

## Purpose
Given an entry URL, discover the set of reachable pages within a bounded
budget and produce a `SiteMap` artefact that downstream agents (Test
Design, Multi-page Runner) can iterate over.

## Scope
- Bounded BFS from the entry URL, configurable `maxDepth`, `maxPages`,
  `maxConcurrency`, `perHostQps`.
- Respect `robots.txt` by default (opt-out flag `--ignore-robots`).
- Same-origin by default; opt-in subdomain following via
  `--include-subdomains`.
- Deduplicate by route pattern (collapse `/posts/1`, `/posts/2`,
  `/posts/3` to one entry tagged `routePattern: "/posts/:id"`).
- Capture per page: URL, title, status code, depth, captured-at,
  reference to the PageAnalysis JSON.
- Persist `SiteMap` to `reports/sitemaps/{crawl-id}.json` and DB row.

## Out of scope
- JavaScript-driven crawling (sitemap.xml ingestion only when fetch
  succeeds; client-routed SPA discovery handled best-effort via
  link extraction after page settles).
- Distributed crawl across multiple machines.
- Headless-detection evasion.

## Acceptance criteria
- AC-009-1: A reachable public site of Nâ‰Ą5 pages produces a SiteMap
  with â‰Ą N - depth-skipped pages within `maxPages` budget.
- AC-009-2: A page disallowed by `robots.txt` is not fetched unless
  `--ignore-robots` is passed; an audit-trail entry records the choice.
- AC-009-3: Route patterns are collapsed: `/posts/1` and `/posts/2`
  produce one SiteMap entry with `routePattern: "/posts/:id"` and a
  representative `sampleUrl`.
- AC-009-4: `perHostQps` is honoured; no requests faster than the
  configured throttle (test by injecting clock).
- AC-009-5: Crawl exit reason is recorded: `maxPagesReached`,
  `maxDepthReached`, `timeoutReached`, `done`.
- AC-009-6: SiteMap is durable: re-running with the same crawl id
  loads from disk without re-fetching.

## Constraints
- All HTTP requests carry a `User-Agent` identifying the framework.
- Cookies / session from `--auth-recipe` (REQ-010) propagate to crawl.
- No outbound AI calls inside the crawler itself (page semantic
  enrichment runs separately in the analyser).

## Inputs
- Entry URL (required)
- `CrawlConfig` (budgets, concurrency, robots flag, scope)
- Optional `storageState` path from REQ-010

## Outputs
- `SiteMap` (Zod-typed) + `reports/sitemaps/{crawl-id}.json`
- DB row in `crawls`
- Per-page references to `reports/evidence/{crawl-id}/{page-hash}/`
