# REQ-013 — Non-Functional Quality Coverage

## Purpose
Measure ISO/IEC 25010 quality characteristics beyond functional
correctness: accessibility (WCAG 2.1 AA), performance (Web Vitals),
basic security (HTTP headers / cookie flags), compatibility (browser
matrix), and i18n (locale-driven runs).

## Scope
### Accessibility (axe-core)
- Wire `@axe-core/playwright` (already in package.json).
- Run `injectAxe` + `checkA11y` per scenario after each scenario's
  final navigation; collect violations into
  `ScenarioResult.accessibilityViolations[]`.
- Tag violations by WCAG level (A / AA / AAA derived from axe impact).
- Configurable severity gates (e.g. fail on `serious` + `critical`).

### Performance (Web Vitals)
- Inject `web-vitals` library on page load; capture LCP, CLS, INP,
  TTFB per page navigation.
- Store on `ScenarioResult.webVitals`.
- Configurable thresholds in `framework.config.yaml`:
  `report.performance.lcpMs`, `clsMax`, `inpMs`, `ttfbMs`.
- Validation check: WARN below threshold, FAIL above ×2 threshold.

### Security headers
- After each `open_page`, capture response headers; validate
  `Content-Security-Policy`, `Strict-Transport-Security`,
  `X-Frame-Options` (or `frame-ancestors`),
  `Set-Cookie` flags (`Secure`, `HttpOnly`, `SameSite`).
- Report missing/weak headers in `ScenarioResult.securityChecks[]`.

### Compatibility (multi-browser)
- `RunnerConfig.browsers: ['chromium','firefox','webkit']` (default:
  `['chromium']` to preserve PS-001 behaviour).
- Run scenarios per browser; tag results with `browser` field.
- HTML report renders a compatibility matrix per scenario.

### i18n
- Accept `--locales en,vi,ja` flag → re-run same suite per locale
  with `Accept-Language` header + page `<html lang>` verification.
- Group results by locale in the report.

## Out of scope
- True load testing (we measure single-user metrics only).
- Penetration testing or DAST.
- Real-device emulation (Playwright virtualised devices only).

## Acceptance criteria
- AC-013-1: Every scenario produces an `accessibilityViolations[]`
  array (may be empty); rendering the HTML report shows violations
  with element, impact, and a remediation hint.
- AC-013-2: Web Vitals capture works; `webVitals` is populated with
  numeric LCP/CLS/INP/TTFB for at least the first page navigation
  per scenario.
- AC-013-3: A page missing CSP produces a `securityChecks[]` entry
  with `severity: 'med'`.
- AC-013-4: Setting `browsers: ['chromium','firefox']` runs each
  scenario twice; results are tagged and the matrix renders correctly.
- AC-013-5: A run with `--locales en,vi` runs the suite twice; results
  group by locale.
- AC-013-6: Performance thresholds can be configured; a slow page
  produces a WARN/FAIL check that surfaces in the report.

## Constraints
- Adding non-functional checks must not regress PS-001 default runtime
  by more than 25% (configurable; defaults can disable expensive
  checks).
- axe + Web Vitals scripts must be served from the local Playwright
  context (no CDN).
- Browser matrix execution is sequential by default (parallelism is a
  Sprint 6 follow-up).

## Inputs
- `RunnerConfig` (browsers, locales, performance thresholds)
- `ReportConfig` for severity gates

## Outputs
- Per-scenario: `accessibilityViolations`, `webVitals`,
  `securityChecks`, `browser`, `locale`.
- Run-level rolls in `RunSummary.nonFunctionalCoverage`.
