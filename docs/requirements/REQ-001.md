# REQ-001 — Page Analyzer

## Purpose
Given a web page URL, open the page in a controlled browser, capture its
DOM/snapshot/screenshot, and produce a structured **PageAnalysis** object
that downstream agents (Test Design, Test Execution) can use.

## Scope
- Open a single URL in a Chromium browser context via Playwright.
- Collect: page title, URL, viewport, screenshot, DOM accessibility tree,
  list of interactive elements (forms, inputs, buttons, links, dropdowns,
  checkboxes, radios, dialogs, tables), required/optional field hints,
  visible error/validation messages, navigation links, dynamic regions.
- Persist the result as `PageAnalysis` JSON keyed by URL + timestamp.
- Re-run analysis on demand and on URL change.

## Out of scope
- Multi-page crawling / sitemap discovery (future REQ).
- Mobile / device emulation (only desktop in MVP).
- DOM mutation tracing during long-running interactions.

## Acceptance criteria (high level)
- AC-001-1: Given a valid public URL, the analyzer returns a non-empty
  PageAnalysis containing at least `title`, `url`, `elements[]`,
  `screenshotPath`, `capturedAt`.
- AC-001-2: Each interactive element in `elements[]` exposes a stable,
  semantic locator (role + name preferred; testId fallback).
- AC-001-3: Required form fields are detected (aria-required, `required`,
  or visual label heuristics) and flagged on the element.
- AC-001-4: A screenshot of the analyzed page is saved under
  `reports/evidence/{run-id}/page.png`.
- AC-001-5: Invalid or unreachable URL surfaces a typed error with a
  human-readable reason; no partial PageAnalysis is persisted.

## Constraints
- Use Playwright Chromium; no Selenium for MVP.
- Respect `robots.txt` only when explicitly configured (default off — test
  environments).
- Sensitive fields (`type=password`, fields tagged PII) must be flagged
  and **not** included in screenshots in clear text where feasible.

## Inputs
- `url: string` (required)
- `viewport?: { width: number; height: number }` (default 1280×800)
- `authState?: StorageStatePath` (optional, for protected pages)

## Outputs
- `PageAnalysis` JSON (see ADR for shape — Phase 3 will define it)
- Screenshot file path
- Evidence directory under `reports/evidence/{run-id}/`
