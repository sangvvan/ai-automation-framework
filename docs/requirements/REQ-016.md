# REQ-016 — Advanced Interactions & Test Data

## Purpose
Cover the long tail of real-world UI interactions and produce
realistic, locale-aware, deterministically-seeded test data without
leaking PII.

## Scope
### New keywords
- `upload_file(target, filePath)` — uses Playwright's `setInputFiles`.
- `drag_drop(source, target)` — Playwright `dragTo`.
- `switch_to_frame(frameLocator)` — set a "current frame" on the
  runner context; subsequent keywords resolve against it; expires at
  scenario end or explicit `switch_to_main`.
- Shadow-DOM piercing in `resolveLocator` when locator carries
  `pierceShadow: true`.
- `type_keyboard(keys)` for keyboard shortcuts (Tab / Enter / Esc).
- `scroll_to(target)` to trigger lazy load.

### Synthetic test data generator
- New module `app/lib/scenario/test-data-gen.ts` using
  `@faker-js/faker` with deterministic seed (`hash(runId + scenarioId)`).
- Heuristic mapping from field name to faker type
  (email → `internet.email`, phone → `phone.number`, etc.).
- Locale-aware (uses run's locale from REQ-013).
- Default placeholder for unrecognised fields stays the current
  `test-${field}` to preserve PS-001 behaviour.
- Mask in reports per existing `mask.ts`.

## Out of scope
- Fuzz generation of malicious payloads (covered minimally in REQ-013
  security check guidance, not here).
- File upload over multiple chunks / drag-drop of huge files.

## Acceptance criteria
- AC-016-1: A scenario calling `upload_file` against an `<input
  type="file">` succeeds and the page reports the file name.
- AC-016-2: `drag_drop` moves a sortable item by one position on a
  fixture page.
- AC-016-3: `switch_to_frame` lets subsequent `click` actions target
  buttons inside an iframe; `switch_to_main` returns control to the
  parent.
- AC-016-4: Shadow-DOM piercing finds an element inside a closed
  Shadow root when the locator carries `pierceShadow: true`.
- AC-016-5: Synthetic data is deterministic: same seed produces the
  same values across runs.
- AC-016-6: Synthetic email values respect locale (e.g. `vi` locale
  uses `.vn` TLDs when available in faker).

## Constraints
- New keywords must follow ADR-005 (no raw `page.locator` / `page.$`
  outside the keywords folder).
- Faker is loaded lazily to avoid bloating cold-start when synthetic
  data is unused.
- Generated PII candidates (names, emails) must never collide with
  values observed on the page (anti-harvest check).

## Inputs / Outputs
- Inputs: scenario step actions + `RunnerConfig.locale`.
- Outputs: scenario can now exercise file upload, iframe content,
  shadow-DOM widgets, and realistic data.
