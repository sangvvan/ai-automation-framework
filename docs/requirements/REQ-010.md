# REQ-010 â€” Authenticated Test Flows

## Purpose
Allow the framework to analyse, crawl, and test protected pages by
performing a one-time login and reusing the resulting browser
`storageState` for all subsequent crawl + scenario operations.

## Scope
- Two ways to specify auth:
  1. **Auto-detect**: framework inspects the login URL with the Page
     Analyzer and infers username/password fields (heuristic on
     `name`/`type`/`label` + presence of submit button).
  2. **Recipe**: tester provides a YAML `auth-recipe.yaml` with
     `loginUrl`, field locators (semantic), `username`, `password`,
     optional MFA stub, optional post-login wait condition.
- Execute login â†’ capture `storageState` â†’ store under
  `reports/evidence/{run-id}/storage-state.json` (mode 0600).
- Reuse storage state for every browser context spawned by the crawler
  and runner during this run.
- Detect session expiry: if a page redirects to login mid-run, re-run
  the recipe once; otherwise fail with `AuthExpiredError`.

## Out of scope
- SSO / OAuth / SAML / WebAuthn flows.
- CAPTCHA solving (recipe may declare `expectsCaptcha: true` â†’ run aborts
  with a clear message).
- Multi-tenant credential vaults.

## Acceptance criteria
- AC-010-1: Auto-detect succeeds on a conventional username/password
  form (one text input + one password input + one submit button).
- AC-010-2: A recipe-driven login succeeds, captures `storageState`,
  and the next page visit shows authenticated content.
- AC-010-3: `storageState.json` permissions are `0600` and the file
  is git-ignored.
- AC-010-4: If credentials are wrong, the framework fails fast with
  `AuthError` referencing the recipe path; no partial run is reported.
- AC-010-5: Mid-run session expiry triggers exactly one re-login;
  a second expiry exits with `AuthExpiredError`.
- AC-010-6: Recipe values reference env vars (e.g. `${SITE_PASSWORD}`);
  raw credentials are never logged.

## Constraints
- Recipes must not embed plaintext passwords â€” `${ENV_VAR}` only.
- Storage state never sent to AI providers.
- Recipe is excluded from `tests/approved/` writes.

## Inputs
- `--auth-recipe inputs/auth/<site>.yaml` (CLI flag)
- Env vars referenced in the recipe

## Outputs
- `storage-state.json` under the run's evidence dir
- DB column `runs.auth_recipe_id` (optional)
- `AuthError` / `AuthExpiredError` typed errors
